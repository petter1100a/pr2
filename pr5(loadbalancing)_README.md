# Practical 5: Load Balancing Simulation

## Theory
**Load Balancing** is the process of distributing network traffic across multiple servers to ensure no single server bears too much demand.

### Algorithms:
1. **Round Robin:** Requests are distributed sequentially to the next server in line.
2. **Random:** Requests are assigned to a server chosen at random.
3. **Least Connections:** Requests are sent to the server with the fewest active sessions.

## How to Run
Run the Python script. It will prompt you to select an algorithm (1, 2, or 3) and simulate 10 requests.
